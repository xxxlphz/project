Plan and design

My project focuses on showing the prevalence of different mental disorders in any specific country, continent or the world over a wide range of years (1990-2019). This information system will serve to inform people about the prevalence of mental disorders in our world by visualising the information using different types of interactive graphs. I will develop this project using a Waterfall approach for the majority of it, prioritisng the completion of each previous phase before moving on in order to create a strong foundation for each step and minimise risks along the way.

Meeting Basic requirements
After obtaining the data from the dataset in the form of a CSV file, the data will be extraced, processed and cleaned in python using Pandas, and all desired information will be stored using Firebase Realtime Database. The cleaned data will be analyzed using python and simple visualisations of the main graphs will be created in the form of line charts and bar graphs by using Plotly. Python structures, namely lists and dictionaries (among others) will be used to manage and store the analyzed data. Using HTML, CSS, Javascript and Chart.js, a web-based interface will be created, displaying the created visualisations in an organised and interactive way.

Meeting Advanced requirements
Each of the graphs will be interactive in multiple ways. This includes a mix of hovvering to see details, modifying the range of years as well as choosing and changing different countries, coninents, disorders and years to edit the information shown on the graph. Furthermore, a survey form will be created, where the user will be prompted to answer questions regarding the prevalence of mental disorders in their own lives. Three or more types of input will be requested from the user. After vlaidation, this information will be stored and processed using Javascript and Firebase, and automatically upgrading graphs will present the user with data from their own survey entry as well as the entries of previous users. Finally, a Q&A (Questions & answers) section will provide users with answers and information to a series of questions crafted using inputs from the user.

My project places heavy emphasis on the brief's values of ethics, integrity and interactivity. Through the use of Html, Python, JS and more, it serves to better peoples understanding of mental disorders in not just our lives, but the lives of people around us.

Problems

The most challenging problem I encountered during development was figuring out how to create the graphs for the survey using the updated data after each submittion, rather than the data that was imported when the webpage first ran. I considered many different ways to solve this, including refreshing the page after every submittion. What I decided to do was to update the value of my variable 'data', which stored everything inside the database, after every submition. I did this by taking another snapshot of the database and then using it to update the variable. Right after, I call my function that makes the graphs and I pass in the updated variable. This allows the graphs to be shown immediately after submition and allows them to include the data from the recent submition

Another problem I experienced was while cleaning my data from my dataset. In the dataset, each row stored the data for a country, continent or the world in a specific year. There was a column that stored the code which corresponds to every country (eg. Afghanistan = AFG, Ireland = IRE). While searching for empty values, I found that countries and world-wide data had codes. Continents and other sub-headings did not. At first, I was stumped. I didn't want to completely disregard the codes as they were useful for identifying countries, but adding codes to the missing values would require me to compile data from outside my dataset. Eventually, I decided to use these missing values to my advantage. I sorted every row with a code into a dictionary that stored the values for each country, and then added the rows That stored the values of each continent into a different dictionary. I uploaded my data to Firebase sorted by year, and then world-wide data, country data and continent data.

Analytics

Below is a pseudocode representation of how I analyzed the (clean) data and prepared it for graphing

  1. IMPORT data from Firebase
  2. CREATE variables (dictionaries, lists) that store data from each subheading inside Firebase
  3. FOR every year:
    1. Extract all unique disorder names, country names, continent names, etc.
    2. Use these to create lists for axes labeling and data categroisation
    3. PRINT the data at each step to verify functionality
  4. Visualise data:
    1. Use sorted data to create charts/graphs in Plotly
  5. UPLOAD the data to Firebase under a unique subheading to be accessed for graphing

Explaination:
For the analytics section of my project, I made an algorithm that store the names of every continent, country, year and disorder in seperate lists. This I incredibly useful for two rasons: the first is using the years and disorders as axes for my bar and line graphs. The second is for accessing the countries and continents I have data on in a quick and easy way. I used this in my website to give the user a list of countries to choose from and to check if they inputted an invalid country/continent. By storing them in seperate lists, the data can be accessed in a fast as efficient way, instead of having to create functions and loops to sort through the original data





Evaluation

In regards to the brief, the project meets all basic and advanced requirements. A dataset was selected and the data was cleaned and stored using Python and Firebase. Analytics were carried out in Python and data from those analysations was used to create a set of basic graphs, with axes and labels, to visually represent the data. a web-based interface was made, and it displayed the data to the user in an interactive way.
Each graph has its own set of interactive features. These include choosing the range, changing the year, country, continets, and showing averages. On a seperate webpage, a survey form was made. The form takes in inputs from the user, validates it and stores the new submition in Firebase alongside previous ones. Two graphs are then created, showing the user information about the collection of data attained from their submittion alongside previous ones. the reccomendations were made in the form of a Q&A section that gives information to the user based on a curated question.

While my project meets all the requirementss, I beleive there is still alot of room for progress. For example, using more graph types, such as pie charts and scatter plots, would help show the user information in more unique ways. Aditionally, allowing the user to search information based on disorder rather than geographic location would make the data much more accessible and easy to navigate.

In the future, I believe adding more information about each chart could help the reader understand the data they are viewing. The content of these descriptions could be set to automatically update depending on what variables the user chooses, for example every time they change the year or choose a new country.
